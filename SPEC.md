# 保険手続きナビ — 実装指示書（Codex 向け）

## 0. 目的・概要

「日本中の保険会社・共済・保険代理店の、各種手続き（解約・給付金請求・各種変更）の連絡先を一括検索できるサイト」を実装する。
ユーザーは **会社名** または **手続きの目的** から、**今すぐ電話できる窓口・電話番号・営業時間・マイページ・公式サイト** にたどり着ける。

完成済みの動作プロトタイプが `codex/reference/insurance-nav-prototype.dc.html` にある（社内フォーマット = Design Component 形式）。
**見た目・挙動・コピーはこのプロトタイプを正とする。** Codex はこれを通常の Web 技術（後述）で再実装すること。データは `codex/data/insurers.json` に全て入っている。

---

## 1. 成果物・技術スタック

- 出力：**単一ページの静的サイト**（SPA で可）。サーバー不要、静的ホスティングで動く構成にする。
- 推奨スタック：**React + Vite + TypeScript**（または Next.js の静的書き出し）。状態管理は標準フックで十分。
- スタイル：任意（CSS Modules / Tailwind / styled-components いずれも可）。**プロトタイプの配色・余白・角丸・影を踏襲**すること。
- データは `insurers.json` を **import して読み込む**（フェッチでもバンドルでも可）。
- **レスポンシブ必須**（スマホ／PC 両対応）。
- 依存は最小限に。重い UI ライブラリは不要。

## 2. データ：`codex/data/insurers.json`

トップレベル構造：

```
{
  meta:        { title, description, totalCompanies, countsByCategory, sources[], notes[], iconSkipHosts[] },
  categories:  [{ key, label, fg?, bg? }],          // 先頭は { key:"all", label:"すべて" }
  procedures:  [{ key, icon, color }],              // 手続きチップ（6種、順番どおり表示）
  hoursPresets:{ <hoursKey>: { text, rows[], open } },
  memos:       { <category>: { <procedureKey>: 案内文(string) } },
  companies:   [ Company, ... ]                     // 88件
}
```

### Company

| フィールド | 説明 |
|---|---|
| `id` | 一意の連番 |
| `name` | 会社名（表示用） |
| `kana` | ひらがな読み（検索用） |
| `category` | `nonlife`(損保) / `life`(生保) / `kyosai`(共済) / `mini`(少額短期) / `agency`(相談・代理店) |
| `brandColor` | ロゴタイルのフォールバック背景色 |
| `initials` | フォールバック表示の頭文字。`\n` は2段表示の改行 |
| `hoursKey` | `hoursPresets` のキー（営業時間） |
| `host` | 公式ドメイン。**空文字 `""` はドメイン不明＝アイコン取得不可** |
| `officialUrl` | 公式サイト（host が無い社は Google 検索 URL） |
| `mypageUrl` | マイページ案内（多くは Google 検索 URL） |
| `phones` | `[{ label, number }]`。**空配列の社は電話番号未掲載** |
| `verified` | `true` = 保険協会の公開データ由来の実電話番号 |

### categories の件数
`nonlife: 29 / life: 41 / kyosai: 5 / mini: 5 / agency: 8`（合計 88）

### procedures（手続きチップ、この順で表示）
`解約` / `給付金・保険金請求` / `住所・氏名変更` / `受取人・払込変更` / `口座振替→クレカ` / `クレカ変更`

### hoursPresets の `open`（営業中判定に使う）
```
open: { weekday:[開始,終了]|null, saturday:[...]|null, sunday:[...]|null, holiday:[...]|null }
```
値は **0:00 からの経過分**（例：`540` = 9:00、`1080` = 18:00）。`null` はその曜日休み。

## 3. 画面構成（プロトタイプ準拠）

1. **ヘッダー（sticky）**：ロゴ「保険手続きナビ」＋説明「日本中の保険会社の、各種手続き連絡先一覧」、右に「{total}社を掲載」バッジ。
2. **ヒーロー（濃紺）**：バッジ → 大見出し「保険手続きナビ」 → 説明文 → **会社名検索ボックス** → 「手続きから探す」**手続きチップ群**。
3. **絞り込みバー（sticky）**：カテゴリタブ（すべて/損害保険/生命保険/共済/少額短期/相談・代理店、各件数つき）＋「営業中のみ」トグル。
4. **結果グリッド**：会社カードを `repeat(auto-fill, minmax(330px, 1fr))` で並べる。並び順は **営業中を先頭**、その後は `id` 昇順。
5. **掲載情報の注意書き**（出典・営業時間は目安・要確認）。
6. **フッター**（出典明記）。
7. **詳細スライドオーバー**（カードの「詳しい窓口・手続き」で右からスライドイン）。

### 会社カード
- 左上：**ロゴタイル**（後述のアイコン方針）。
- 会社名＋かな、右上に **営業中/時間外バッジ**（営業中は緑＋点滅ドット）。
- カテゴリバッジ（categories の `fg`/`bg`）＋営業時間テキスト（`hoursPresets[hoursKey].text`）。
- **強調窓口ボックス**：
  - 手続きチップ未選択時 → 先頭の電話（`phones[0]`）を大きく表示。電話が無い社は「公式サイトで番号を確認」リンク。
  - 手続きチップ選択時 → その手続きに最適な窓口を表示（`pickPhone` ロジック、後述）＋見出しを「『{手続き}』の窓口：{ラベル}」に。
- フッターボタン：**詳しい窓口・手続き**（詳細を開く）／**マイページ**（`mypageUrl`）。

### 詳細スライドオーバー
- ヘッダー：ロゴタイル＋会社名＋かな＋カテゴリ＋営業中バッジ。
- **問い合わせ窓口**：`phones` を全件リスト（`tel:` リンク）。空なら「ご契約のしおり・証券／公式サイトでご確認ください」＋公式リンク。
- **営業時間（目安）**：`hoursPresets[hoursKey].rows` を表で表示。
- **手続き別のご案内**：`procedures` 6種それぞれについて、`memos[category][procedureKey]` の案内文＋推奨窓口（`pickPhone` のラベル）をカード表示。アイコン・色は `procedures[].icon/color`。
- フッター：マイページ／公式サイトボタン。

## 4. ロジック詳細

### 4.1 検索
`query` を会社名（`name`）・かな（`kana`）に部分一致（小文字化／かなはそのまま含有判定）。

### 4.2 カテゴリ絞り込み
`category === 選択キー`（`all` は全件）。タブの件数は **検索適用後・営業中フィルタ適用前** の件数。

### 4.3 営業中フィルタ＆判定（`isOpen`）
- **日本時間（Asia/Tokyo）** の曜日・時刻で判定する。
  ```js
  const p = new Intl.DateTimeFormat('en-US', {timeZone:'Asia/Tokyo', weekday:'short', hour:'2-digit', minute:'2-digit', hour12:false}).formatToParts(new Date());
  // weekday(Sun..Sat) と 分(hour*60+minute) を得る
  ```
- `hoursPresets[hoursKey].open` から、該当曜日の `[開始,終了]` を引き、`開始 <= 分 < 終了` なら営業中。
- 曜日マッピング：日=sunday / 月〜金=weekday / 土=saturday。**祝日は簡易実装で可**（`holiday` ルールがあれば使う。なければ平日扱いでよい。プロトタイプも祝日は厳密判定していない）。
- 「営業中のみ」トグル ON で営業中の社だけ表示。

### 4.4 `pickPhone(company, procedureKey)`
1. `phones` が空なら `null`（→「公式サイトで番号を確認」）。
2. `procedureKey === '給付金・保険金請求'` のとき、ラベルに `保険金|支払|請求|給付` を含む電話があればそれを優先。
3. それ以外は `phones[0]`。

### 4.5 強調窓口の補足テキスト（プロトタイプ準拠）
- `nonlife` かつ `給付金・保険金請求` 選択時 → 「事故受付は24時間365日対応の場合があります」。
- それ以外 → 営業中なら「ただいま受付中です」、時間外なら「営業時間外（{hours.text}）」。

## 5. アイコン方針（重要）

各社の **公式サイトのファビコン/ロゴ** を外部サービスから取得し、白い角丸タイル（アプリ風）で表示する。

- `host` が空、または `meta.iconSkipHosts` に含まれるドメインは **取得しない** → `brandColor` 背景＋`initials`（`\n` で2段）のタイルを表示。
- 取得は複数ソースのフォールバックを推奨（プロトタイプ実績）：
  1. `https://www.google.com/s2/favicons?sz=128&domain={host}`（実用上ほぼこれが表示される）
  2. `https://icons.duckduckgo.com/ip3/{host}.ico`
  3. 失敗時は `<img>` を隠し、下層の `brandColor`+`initials` タイルを見せる（`onError` で `display:none`）。
- `iconSkipHosts`（例：`justincase.jp`）は**別社のロゴが返るため**自動取得しない。**新たに誤ロゴが見つかったらこの配列に追記**する運用。
- ⚠ ロゴ／ファビコンは各社の著作物。本サイトでは「リンク先の識別」目的の小サイズ表示に留める。問題があれば該当社を `iconSkipHosts` 化する。

## 6. デザイン指針（プロトタイプ準拠）

- フォント：見出し `Zen Kaku Gothic New`（700/900）、本文 `Noto Sans JP`。
- 配色：ページ背景 `#eef1f6` / カード白 / 文字 `#0d1626` / ミュート `#5d6b80` / 罫線 `#e5eaf2`。
- アクセント（ボタン・電話番号・リンク）：既定 `#2456e6`。ヒーローは濃紺 `radial-gradient(1100px 520px at 72% -12%, #16335f, #0b1b34)`。
- 営業中：緑 `#19a06a`。電話番号は太字・等幅数字（`font-variant-numeric: tabular-nums`）でタップ発信（`tel:`）。
- トーン：**公的・専門的で信頼感のある**実務系。絵文字・過度なグラデは使わない。
- ヒット領域はスマホで 44px 以上。スライドオーバーは `min(480px, 100%)`。

## 7. 注意・免責（サイトに必ず明記）

- 出典：**生命保険協会（2026/4/13 時点）／日本損害保険協会（2026/4/1 時点）** の会員会社一覧の公開情報。`life`・`nonlife` の社名・窓口・電話・公式サイトはこれに基づく。
- `kyosai`・`mini`・`agency` は公式サイトリンク中心、電話は未掲載。
- **全社の営業時間は代表的な目安。** 窓口・受付時間は商品・手続きで異なる。
- 利用者には「**お手続き前に必ず各社公式サイト・ご契約書類で最新情報を確認**」と案内する。
- `agency`（代理店）は引受保険会社へのお取次ぎ窓口。解約・給付金の最終手続きは引受保険会社で行う旨を `memos.agency` に記載済み。

## 8. 今後の拡張（実装時に壊さない設計に）

- `phones` 未掲載の社（共済・少短・代理店）に実電話番号を後から追加できる構造を維持。
- 営業時間は `hoursPresets` を増やす／各社個別時間を持たせる拡張が可能なように。
- 五十音／カテゴリ別インデックス、事故受付24時間ダイヤルの個別掲載は将来追加候補。
