## ファイル
- `SPEC.md` … 実装指示書（**まず最初に読む**）。要件・データ仕様・ロジック・デザイン・受け入れ基準。
- `data/insurers.json` … 全データ（88社＋カテゴリ／手続き／営業時間プリセット／手続き別案内文）。これを import して使う。
- `reference/insurance-nav-prototype.dc.html` … 完成済みの**動作プロトタイプ**（社内フォーマット）。見た目・挙動・コピーの正。ブラウザで開けば動作を確認できる。

## 進め方
1. `SPEC.md` を読む。
2. `reference/insurance-nav-prototype.dc.html` をブラウザで開き、UX とデザインを確認。
3. `data/insurers.json` の構造を把握。
4. React + Vite + TypeScript（または同等）で静的サイトとして再実装。
5. `SPEC.md` 第9章のチェックリストで検証。
